# DeskPilot 需求规格书

> 本文档是 DeskPilot Windows 桌面整理工具的完整需求规格。

---

## 1. 产品定位

DeskPilot 是一个轻量级 Windows 桌面整理工具。

### 核心目标

1. 自动整理 Windows 桌面文件
2. 支持自然语言整理需求
3. 将飞书待办任务显示到桌面
4. 将飞书近期日程提醒显示到桌面
5. 支持直接在桌面添加飞书待办
6. 支持独立主窗口、桌面悬浮组件、系统托盘三种形态
7. 保持极低 CPU 和内存占用
8. 支持深色 / 浅色两套皮肤
9. 支持开机自启动

### 产品关键词

轻量 · 简洁 · 安全 · 高级 · 克制 · 可控 · 本地优先 · AI 按需触发

---

## 2. 技术栈要求

### 必须使用

- .NET 8
- WPF
- MVVM 架构
- C#
- xUnit 或 NUnit
- Windows 原生托盘能力
- Windows 开机启动能力
- Windows Toast 通知能力

### 不使用

- Electron
- 常驻 AI Agent
- 后台持续文件扫描
- 复杂动画框架
- 云端文件存储

### 推荐项目结构

```
DeskPilot/
  src/
    DeskPilot.App/           # WPF 主应用
    DeskPilot.Core/          # 核心业务逻辑
    DeskPilot.Infrastructure/ # 基础设施
    DeskPilot.Tests/         # 单元测试
  docs/
    interaction/
    visual/
    architecture/
  README.md
```

## 4. 文件整理范围

### 4.1 整理目标

MVP 阶段只整理 Windows 桌面目录。

**默认桌面路径：**
```
%USERPROFILE%\Desktop
```

**兼容 OneDrive 桌面路径：**
```
%USERPROFILE%\OneDrive\Desktop
```

程序启动时需要识别真实桌面路径。

### 4.2 默认不处理的文件

1. 系统图标
2. 回收站
3. 应用快捷方式（`.lnk`）
4. 隐藏文件
5. 系统文件
6. 正在被占用的文件
7. 文件夹本身（除非用户明确要求）
8. 程序自身生成的日志文件

## 5. 整理触发方式

**只做用户主动触发。**

支持的触发方式：
1. 一键整理
2. 自然语言整理

**不做：**
1. 后台自动整理
2. 桌面文件变化自动整理
3. 开机后自动整理
4. 定时自动整理
5. 未经确认直接移动文件

## 7. 自然语言整理能力

### 7.1 示例需求

```
把最近和项圈试产相关的文件整理到一个文件夹里，图片和文档分开。
把今天创建的图片和文档分开整理。
把桌面上的压缩包和安装包放到临时文件夹。
把和飞书应用部署相关的资料整理到项目文件夹。
把最近 7 天下载到桌面的文件按项目分组。
```

### 7.2 AI Provider 配置

支持 OpenAI-Compatible API。

**配置项：**
- Base URL
- API Key
- Model Name
- Temperature
- Timeout
- Max Tokens
- 是否启用 AI 整理

**要求：**
1. API Key 本地安全存储
2. 不写入明文日志
3. 网络失败时明确提示
4. 超时后自动终止请求
5. AI 不可用时降级为规则整理

### 7.3 AI 输入数据

**允许发送：**
1. 文件名
2. 扩展名
3. 文件大小
4. 创建时间
5. 修改时间
6. 相对路径

**默认不发送：**
1. 文件正文
2. 图片内容
3. PDF 内容
4. Word 内容
5. Excel 内容
6. 绝对路径中的用户名
7. 系统完整路径
8. 文件内容摘要

### 7.4 AI 输出要求

AI 必须输出结构化 JSON 整理计划。

**示例输出：**
```json
{
  "foldersToCreate": ["项圈试产", "项圈试产/图片", "项圈试产/文档"],
  "moves": [
    {
      "source": "项圈试产进度.png",
      "target": "项圈试产/图片/项圈试产进度.png",
      "reason": "文件名包含项圈试产，且为图片文件"
    }
  ],
  "skipped": [
    {
      "file": "Chrome.lnk",
      "reason": "快捷方式默认不处理"
    }
  ],
  "warnings": ["本次不会删除任何文件"]
}
```

**AI 输出非法 JSON 时：**
1. 不执行整理
2. 展示错误提示
3. 保留原始错误日志
4. 提供"重试"和"改用规则整理"

## 8. 整理安全机制

### 8.1 整理流程

所有整理必须满足：

1. 先扫描
2. 再生成计划
3. 再预览
4. 用户确认后才执行
5. 不删除文件
6. 不覆盖文件
7. 不移动系统文件
8. 不处理快捷方式
9. 文件被占用时跳过
10. 每次整理生成操作日志
11. 支持撤销上一次整理

### 8.2 整理计划预览

预览页必须显示：

1. 将创建的文件夹
2. 将移动的文件
3. 原位置
4. 目标位置
5. 跳过文件
6. 跳过原因
7. 风险提示
8. 执行按钮（"确认整理"）
9. 取消按钮

### 8.3 同名文件处理

禁止覆盖。

如果目标目录存在同名文件，自动改名：
```
文件名.ext
文件名 (1).ext
文件名 (2).ext
```

### 8.4 撤销机制

**撤销内容包括：**
1. 原路径
2. 目标路径
3. 创建的文件夹
4. 实际移动成功的文件
5. 跳过的文件
6. 失败原因

**撤销规则：**
1. 只撤销上一次整理
2. 文件不存在时跳过并提示
3. 目标路径被占用时跳过并提示
4. 不删除用户后来新增的文件
5. 只删除本次整理创建且为空的文件夹

## 9. 飞书接入

### 9.1 lark-cli 基础要求

通过子进程调用 `lark-cli`，封装为 `LarkCliService`。

**需要支持：**
1. 检查 lark-cli 是否安装
2. 检查 lark-cli 版本
3. 检查 lark-cli 登录状态
4. 触发用户扫码 / 浏览器授权登录
5. 检查 calendar 和 task 权限
6. 获取待办
7. 新增待办
8. 完成待办
9. 获取日程
10. 检查 lark-cli 更新
11. 引导用户更新 lark-cli

### 9.2 登录方式

**要求：**
1. 首次使用时展示飞书连接引导
2. 点击"连接飞书"后调用 lark-cli 登录
3. 展示 lark-cli 返回的授权链接或二维码入口
4. 用户完成授权后检查登录状态
5. 登录成功后展示当前用户身份
6. 登录失败时展示失败原因
7. 支持重新登录
8. 支持退出登录

### 9.3 权限范围

**至少需要：**
1. task 相关权限
2. calendar 相关权限

**如果权限不足：**
1. 不崩溃
2. 展示缺失权限
3. 提供重新授权按钮
4. 不展示空白错误页

### 9.4 CLI 输出要求

**要求：**
1. 所有读取命令优先使用 JSON 输出
2. 解析 stdout JSON
3. 捕获 stderr
4. 捕获 exit code
5. 设置超时时间
6. 支持取消
7. 所有写操作优先 dry-run 或二次确认
8. Windows 下必须处理参数转义问题
9. 不把 token、secret、授权信息写入日志

### 9.5 飞书待办

**显示范围：** 未来 14 天待办

**优先级排序：**
1. 逾期未完成
2. 今天到期
3. 明天到期
4. 未来 14 天内任务

**支持操作：**
1. 查看待办标题
2. 查看到期时间
3. 查看完成状态
4. 新增待办
5. 完成待办
6. 手动刷新
7. 定时刷新

**新增待办位置：**
```
桌面组件 > 今日待办区域底部输入框
```

**输入行为：**
1. 输入内容
2. 回车创建
3. 创建成功后清空输入框
4. 创建失败保留内容并提示原因

### 9.6 飞书日程

**默认展示：**
1. 今日日程
2. 明日日程

**支持切换查看：**
- 未来 7 天日程

**日程字段：**
1. 标题
2. 开始时间
3. 结束时间
4. 地点
5. 会议链接
6. 参与人数量（可选）
7. 日程状态（可选）

**提醒：**
1. 日程开始前提醒
2. 默认提前 10 分钟
3. 用户可关闭提醒
4. 不重复提醒同一日程

### 9.7 刷新频率

**默认：**
1. 程序启动后立即刷新一次
2. 待办每 10 分钟刷新一次
3. 日程每 10 分钟刷新一次
4. 用户可手动刷新

**要求：**
1. 空闲时不频繁请求
2. 网络失败不弹连续错误
3. 最近一次刷新时间需要展示
4. 刷新失败保留上一次成功数据

## 11. 性能要求

### 11.1 轻量化目标

1. 空闲状态 CPU 接近 0%
2. 空闲状态内存占用尽量低
3. 不做后台 AI 推理
4. 不持续扫描桌面
5. 不持续监听文件变化
6. 不使用 Electron
7. 不做复杂动画
8. 飞书数据按 10 分钟刷新
9. 文件扫描只在用户触发整理时执行
10. 退出程序后不保留后台进程

### 11.2 性能目标

| 指标 | 目标 |
|------|------|
| 空闲 CPU | 小于 1% |
| 空闲内存 | 小于 150MB |
| 500 文件扫描 | 2 秒内完成 |
| UI 操作响应 | 小于 200ms |
